# AI Frontier Industry Knowledge

这是一个面向 AI 前沿行业知识共建的项目仓库，用来沉淀可复用的项目案例、技术资料、行业知识、数据处理流程和实践经验。

项目希望逐步发展成一个开放交流社区：不是一个人把所有内容写完，而是让更多成员把自己验证过的知识、代码、文档和项目经验贡献进来。为了让不熟悉 GitHub 的成员也能参与，仓库采用 **Fork + Pull Request** 的协作方式。

## 如何参与共建

最短路径如下：

1. 在 GitHub 上 Fork 本仓库到自己的账号。
2. 把自己的 Fork 仓库克隆到本地。
3. 基于主仓库的 `dev` 分支创建个人任务分支。
4. 修改文档、代码、数据说明或项目资料。
5. 提交到自己的 Fork 仓库。
6. 向主仓库的 `dev` 分支发起 Pull Request。
7. 等待管理员或模块负责人评审，通过后合并。

完整操作说明见 [CONTRIBUTING.md](./CONTRIBUTING.md)。
参考视频：https://www.bilibili.com/video/BV1bwcWzAE3R/?spm_id_from=333.337.search-card.all.click&vd_source=95bc3e5a1cd7bd1225d9ea9b80761067

可以贡献的内容包括：

- 新增 AI 行业项目案例或项目复盘。
- 补充已有项目的 README、部署说明、接口说明、使用教程。
- 修正文档错误、错别字、路径失效、描述不清的问题。
- 提交可复用的脚本、工具、数据清洗流程或评测方法。
- 在 Issues 中提出问题、建议、任务拆分和资料线索。

## 协作原则

主仓库只保留少量公共分支，普通成员不要直接向主仓库推送代码。每个人在自己的 Fork 仓库中自由修改，通过 Pull Request 把成果交给主仓库评审。

可以把主仓库理解为“社区公共书架”，个人 Fork 仓库理解为“自己的草稿本”。草稿本里可以反复修改，公共书架上的内容则需要经过检查后再放上去。这样既能鼓励大家参与，也能保证主仓库长期稳定、清晰、可维护。

## 仓库结构

```text
.
├── documents/   # 通用文档、资料沉淀
├── projects/    # 具体项目案例和项目代码
└── README.md    # 项目首页
```

## 新手入口

如果还不会 Fork，可以先看 GitHub 官方文档：[Fork a repo](https://docs.github.com/en/get-started/quickstart/fork-a-repo)。
